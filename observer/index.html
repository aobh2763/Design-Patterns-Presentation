Situation

  • Multi-process architecture.
  • A bunch of handlers linked up by queues.
  • Each handler must receive the request.

Proposed Solution

Using Chain of Responsibility

Proposed Solution

  • The order in which the handlers are placed doesn't matter.
  • The handlers are tasked with forwarding the request and handling it, breaking the single responsibility principle.
  • We can decouple these two responsibilities.

Proposed Solution

The intent is completely different from Chain of Responsibility!

Observer

Intent

Observer is a behavioral design pattern that lets you define a subscription mechanism to notify multiple objects about any events that happen to the object they're observing.

Intent

It defines a one-to-many dependency between objects so that when the one object changes state, all its dependents are notified and updated automatically.

Structure

  • The oberserved object notifies the subscribers when an event happens (its state changes).
  • The observed object knows all of the observers.
We'll call it the "Observable".

Structure

Structure of Observable

Structure

  • The observing objects are completely independant.
  • However, they share the same functionality : react when notified by the object they're observing.
  • They must all implement a shared interface.
We'll call them the "Observers".

Real World Applications

GUI Event Listeners

Buttons, sliders, or text fields notify multiple listeners when their state changes (a button click triggers multiple UI updates, for example).

Game Development

Entities such as enemies observe player actions; when the player moves or attacks, observers react accordingly.

Real-Time Logging Systems

Various logging outputs (such as the console or files) subscribe to application events to record them in real-time.

Implementation

                                
import java.util.ArrayList;
import java.util.List;

public class Stock {
    private List<Observer> subscribers = new ArrayList<>();
    private String name;
    private float price;

    public Stock(String name, float price) {
        this.name = name;
        this.price = price;
    }

    public void subscribe(Observer obs) {
        subscribers.add(obs);
    }

    public void unsubscribe(Observer obs) {
        subscribers.remove(obs);
    }

    public void notifySubscribers() {
        for (Observer subscriber : subscribers) {
            subscriber.update(this);
        }
    }

    public void setPrice(float newPrice) {
        // Notify the observers only if the price changes
        if (this.price != newPrice) {
            this.price = newPrice;
            notifySubscribers();
        }
    }

    public float getPrice() {
        return this.price;
    }
}

Implementation

                                
public interface Observer {
    public void update(Stock context);
}

public class TableFacade implements Observer {
    private Table table;

    @Override
    public void update(Stock context) {
        // Update the Table
        table.updateTable(context);
    }
}

public class NotificationFacade implements Observer {
    private Notifier notifier;

    @Override
    public void update(Stock context) {
        // Send a notification if a certain limit is exceeded
        if (context.getPrice() >= PRICE_CEILING) {
            notifier.sendNotification();
        }
    }
}

public class GraphingFacade implements Observer {
    private Graph graph;

    @Override
    public void update(Stock context) {
        // Update the graph
        graph.updateGraph(context);
    }
}

import okhttp3.*;

public class Main {
    static Stock stock = new Stock("StockA", 50.4f);

    public void setupAPI() {
        OkHttpClient client = new OkHttpClient();

        // Building the request
        Request request = new Request.Builder()
                .url("https://realstock.com/api")
                .build();

        // Defining the listener's behavior
        EventSourceListener listener = new EventSourceListener() {
            @Override
            public void onEvent(EventSource source, String id, String type, String data) {
                float newPrice = Float.parseFloat(data);
                stock.setPrice(newPrice);
            }
        };

        // Build the source, send the request and link it to the listener
        EventSources.createFactory(client).newEventSource(request, listener);
    }

    public static void main(String[] args) {
        // Setup observers
        Table table = new Table();
        Notifier notifier = new Notifier();
        Graph graph = new Graph();

        stock.subscribe(new TableFacade(table));
        stock.subscribe(new NotificationFacade(notifier, 150f));
        stock.subscribe(new GraphingFacade(graph));

        setupAPI();

        Application app = new Application();
        app.run();
    }
}

Implementation of the Observers

Benefits

  • Open/Closed Principle
    • You can introduce new observer classes without having to change the observable's code.
  • Single Responsibility Principle
    • Each observer handles its own reaction to the update.
    • The observable only manages state and notifications.

Limitations

  • Observers are notified in random order.
  • The observable doesnâ€™t know its observers, but their reactions might indirectly depend on it.
